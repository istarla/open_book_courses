## What is HTML? 

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It's not a programming language, but rather a markup language that uses tags to structure content on the web. Think of HTML as the skeleton of a webpage - it provides the basic structure and content that browsers can interpret and display.

HTML works by using a system of tags to mark up text, images, and other content to display in a web browser. These tags tell the browser how to structure and present the content to users.


```html
<!DOCTYPE html>
<html>
<head>
    <title>My First Web Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML document.</p>
</body>
</html>
```

Key characteristics of HTML:
- **Markup Language**: Uses tags to annotate content
- **Platform Independent**: Works across different operating systems and devices
- **Case Insensitive**: HTML tags can be written in uppercase or lowercase (lowercase is preferred)
- **Structural**: Focuses on content structure rather than presentation

## A Brief History of HTML 

Understanding the evolution of HTML helps us appreciate its current capabilities and design decisions.

### HTML 1.0 (1993)
The first version of HTML was created by Tim Berners-Lee at CERN. It included basic tags like:

```html
<h1>Heading</h1>
<p>Paragraph</p>
<a href="link">Anchor</a>
<ul><li>List items</li></ul>
```

### HTML 2.0 (1995)
Introduced forms and additional formatting options:

```html
<form action="/submit" method="post">
    <input type="text" name="username">
    <input type="submit" value="Submit">
</form>
```

### HTML 3.2 (1997)
Added support for tables, applets, and text flow around images:

```html
<table>
    <tr>
        <td>Cell 1</td>
        <td>Cell 2</td>
    </tr>
</table>
```

### HTML 4.01 (1999)
Introduced stylesheets, scripting, and better accessibility:

```html
<div style="color: blue;">Styled content</div>
<script type="text/javascript">
    alert("JavaScript support!");
</script>
```

### XHTML (2000)
A stricter, XML-based version of HTML:

```xml
<p>All tags must be properly closed</p>
<br />
<img src="image.jpg" alt="Description" />
```

### HTML5 (2014)
The current standard with semantic elements, multimedia support, and APIs:

```html
<header>
    <nav>
        <ul>
            <li><a href="#home">Home</a></li>
        </ul>
    </nav>
</header>
<main>
    <article>
        <section>
            <h2>Article Title</h2>
            <p>Article content...</p>
        </section>
    </article>
</main>
<footer>
    <p>&copy; 2024 My Website</p>
</footer>
```

## Basic Structure of an HTML Document 

Every HTML document follows a standard structure that browsers expect. Understanding this structure is crucial for creating valid, well-formed web pages.

### The DOCTYPE Declaration

The DOCTYPE declaration tells the browser which version of HTML the document is using:

```html
<!DOCTYPE html>
```

This declaration:
- Must be the very first line in your HTML document
- Is not case-sensitive
- Tells the browser to use HTML5 standards
- Prevents the browser from entering "quirks mode"

### The HTML Element

The `<html>` element is the root element that wraps all content:

```html
<!DOCTYPE html>
<html lang="en">
    <!-- All other content goes here -->
</html>
```

The `lang` attribute:
- Specifies the language of the document
- Helps screen readers and search engines
- Common values: "en" (English), "es" (Spanish), "fr" (French)

### The Head Section

The `<head>` contains metadata about the document:

```html
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="A brief description of your page">
    <meta name="keywords" content="HTML, CSS, JavaScript">
    <meta name="author" content="Your Name">
    <title>Page Title - Shows in Browser Tab</title>
    <link rel="stylesheet" href="styles.css">
    <link rel="icon" href="favicon.ico" type="image/x-icon">
</head>
```

Important head elements:
- **charset**: Defines character encoding (UTF-8 is standard)
- **viewport**: Controls layout on mobile devices
- **title**: Appears in browser tab and search results
- **meta description**: Used by search engines
- **link**: References external resources like CSS files

### The Body Section

The `<body>` contains all visible content:

```html
<body>
    <header>
        <h1>Website Title</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section id="home">
            <h2>Welcome to Our Website</h2>
            <p>This is the main content area.</p>
        </section>
        
        <section id="about">
            <h2>About Us</h2>
            <p>Information about the company or website.</p>
        </section>
    </main>
    
    <aside>
        <h3>Sidebar Content</h3>
        <p>Additional information or links.</p>
    </aside>
    
    <footer>
        <p>&copy; 2024 Your Website Name. All rights reserved.</p>
    </footer>
</body>
```

### Complete Basic Structure

Here's a complete, minimal HTML5 document:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Web Page</title>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <p>This is a paragraph of text on my web page.</p>
</body>
</html>
```

## Tools You Need

### File Management

#### Folder Structure
Organize your HTML projects properly:

```
my-website/
├── index.html
├── css/
│   └── styles.css
├── js/
│   └── script.js
├── images/
│   ├── logo.png
│   └── background.jpg
└── pages/
    ├── about.html
    └── contact.html
```

#### File Naming Conventions
Follow these best practices:

```html
<!-- Good file names -->
<link rel="stylesheet" href="css/main-styles.css">
<img src="images/hero-image.jpg" alt="Hero image">
<a href="pages/about-us.html">About Us</a>

<!-- Avoid spaces and special characters -->
<!-- Bad: "my styles.css", "about us.html" -->
<!-- Good: "my-styles.css", "about-us.html" -->
```
